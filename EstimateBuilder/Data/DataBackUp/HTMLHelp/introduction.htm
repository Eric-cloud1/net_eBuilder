Introduction

 

Using the word “Object” in the name is based on the fact that the estimating application will not be limited to the one industry i.e. construction. The program is Table driven and the users’ soft coded business formulas. The values for each takeoff item will be stored and used by the stored formulas, the program will generate the quantity from the soft coded business formulas, this gives the application its’ great flexibility. The following Relationship Model shows the flow of information, the Tables can be stored on the Local Desktop, Local Server and/or Remote Server, or any combination
