Purpose

 

To create stored values in a record that can be accessed by the Application repeatedly, using the users’ soft coded business formulas. Estimating programs of today do not use such a dynamic database; they may have some or they may even have most the elements of this program but not the flexibility i.e. the random access and the use of the data values repeatedly from both the local and remote files (WEB Sever) systems.

Create a WEB database so the information can be access by multiple users. Local Vendors are charged a fee that will be paid to the WEB Site Host for Advertising and User List from each estimate. The vendors will maintain their stored Costs at will so the information is always current. The vendor’s names and contact info will be on the web site itself as advertising and in the user’s reports. There will be an info sheet at the end of each estimate with the contact info for each of the vendors that will be needed to complete the scope of work for that estimate.

The Webs’ stored data will be used for the Statistical Analysis. We will be able without the violation of the users name or bid info, tell the user how there compare to other users of similar projects in that area or nation wide by any number of factors, this will be a unique tool for the user, giving them the information on how competitive their bid will be.
